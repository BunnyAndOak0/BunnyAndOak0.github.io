用户管理

基本介绍

linux是一个多用户多任务的系统

linux用户至少需要属于一个组

操作

  1. 添加用户

    useradd [选项] 用户名（不指明的话会创建一个和用户同名的组放在home家目录下面）

    例如：

    adduser admin

    也可以使用

    useradd -d 指定目录 新用户名称

    给创建的用户指定家目录（不要先创建目录，等创建用户的时候一起创建）

  1. 给指定用户修改密码

    passwd 用户名

    passwd  admin    //之后输入新密码即可

  2. 删除用户

    • userdel 用户名

      删除用户，但是保留家目录

    • userdel -r 用户名

    但是一般来说，实际中，都会保留家目录

  1. 查询用户

    id 用户名

    id root

    如果用户不存在，会返回无此用户

  1. 切换用户

    su - 切换用户名

    （在linux中，如果当前用户的管理权限不够，可以通过su - 指令，切换到高权限用户）

    su  -  sys1

    从高权限到低权限不需要密码，反之需要

    当需要回到原来用户时，使用exit指令

  2. 查看当前用户

    whoami

    whoami

  2. >指令和>>指令

    >输出重定向和>>追加

    >输出重定向：会将原来文件的内容覆盖

    >>追加：不会覆盖原来文件的内容，而是追加到文件的尾部

    ls -l > 文件 功能描述：列表的内容写入到文件a.txt中（覆盖写）

    ls -al >> 文件 功能描述：列表的内容追加到文件aa.txt的末尾

    cat 文件1 > 文件2 功能描述：将文件1的内容覆盖到文件2

    echo “内容” >> 文件 功能描述：将内容追加到文件2

    （被写入/追加的文件可以不存在）

    ls -l > aaa.txt

ll >> aaa.txt

echo "hello world" >> aaa.txt				//将内容进行追加

echo “hello world” > aaa.txt //将内容进行覆盖

权限的基本介绍

基本介绍

文件各个部分的含义：

rwx权限

rwx作用到文件

  1. r代表可读（read）：可以读取查看
  2. w代表可写，可以修改，但是不代表可以删除该文件，删除一个文件的前提条件是对该文件所在目录有写权限，才可以删除该文件
  3. x代表可执行（execute）：可以被执行

rwx作用到目录

  1. r代表可读（read）：可以读取，ls查看目录内容
  2. w代表可写（write）：可以修改，目录内创建 + 删除 + 重命名目录
  3. x代表可执行（execute）：可以进入该目录

也可用数字表示：r = 4，w = 2，x = 1，因此rwx = 4 + 2 + 1 = 7

修改权限

  1. 修改权限-chmod

  2. 方式一：+、-、=变更权限

    g：所在组

    o：其他人

    a：所有人（u、g、o的总和）

    1. chmod u=rwx, g=rx, o=x 文件目录名

    2. chmod o+w 文件目录名（增加权限）

    3. chmod a-w 文件目录名（减少权限）

      chmod u=rwx,g=rx,o=rw abc	

chmod u-x,g+w abc

chmod 755 /home/abc //用数字表示权限

linux分区

Linux来说，无论有几个分区，分给哪一个目录使用，它归根结底就是有一个根目录，一个独立而且唯一的文件结构，Linux中每个分区都是用来组成文件系统的一部分

Linux采用一种叫做”载入”的处理方法，它的整个文件系统种包含了一整套的文件和目录，而且将一个分区环绕一个目录联系起来，这时要载入的一个分区将使它的存储空间在一个目录下获得

Linux硬盘分为IDE硬盘和SCSI硬盘，目前基本上使SCSI硬盘。

lsblk -f			//查看系统的分区和挂载的情况
  1. 对于IDE硬盘，驱动器标识符为”hdx“，其中”hd”表明分区所在的设备类型，这里是指IDE硬盘，”x”为盘号（a为基本盘，b为基本从属盘，c为辅助主盘，d为辅助从属盘），”“代表分区，前四个分区用数字1-4表示，他们是主分区或者扩展分区，从5开始就是逻辑分区，例如：hda3代表第一个IDE硬盘上的第三个主分区或者扩展分区。
  2. 对于SCSI硬盘则标识为”sdx~”，SCSI硬盘使用”sd”来表示分区所在设备的类型的，其余则和IDE硬盘的表示方法一样。

实操篇—进程管理

进程的基本介绍

在linux种，每一个执行的程序（代码）都成为一个进程，每一个进程都分配一个ID号。

每一个进程，都会对应一个父进程，而这个父进程可以复制多个子进程，例如：www服务器

每一个进程都可能以两种方式存在，前台与后台，所谓前台进程就是用户目前的屏幕上可以进行操作的，后台进程则是实际在操作的，但是由于频幕上无法看到的进程，通常使用后台的方式执行。

一般系统的服务都是以后台进程的方式存在的，而且都会常驻在系统中，直到关机才结束
